## 2. 保留与重写边界

本次迁移不是简单替换品牌名称。为控制工程风险，保留了通用的四层代码组织思想、Pydantic 数据模型方式、状态图/checkpoint、trace、离线网关与测试框架等不绑定具体行业的结构；所有对外文案、品牌与品类数据、领域词典、趋势信号、候选生成、用户验证问题、商品评估、八维评分、决策阈值、API 视图、运行报告和工作台信息架构均按兴趣消费命题重新设计。

重写后的核心业务链路为：公开或获授权证据进入统一 Evidence 层，趋势、需求和竞品白空间三条路径并行生成候选，再按用户验证、商品风险和八维确定性评分排序，最后通过 HITL 形成提案。评分维度为趋势匹配、需求强度、差异化、社交传播、成本与毛利、供应链可行性、IP/合规、全球本地化；这些内容不沿用原命题的行业术语或业务结论。
