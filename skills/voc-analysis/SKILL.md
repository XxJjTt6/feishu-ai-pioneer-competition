# 兴趣消费 VOC 分析

## 适用场景

- 分析 MINISO 与可比兴趣消费品牌的评论、门店反馈或经授权研究数据。
- 为趋势、用户需求、竞品白空间三条候选路径提供结构化机会，而不是直接宣称某个商品会成为爆款。
- 在真实数据尚未接入时，使用固定种子合成样本验收工程链路。

## 数据入口与边界

1. **公开信息**：记录原始 URL、日期、抓取或导入方式和允许的用途；公开可见不等于允许批量抓取或商用。
2. **企业数据**：只接入已获授权字段，执行最小化、脱敏、访问控制、留存期限和删除策略。
3. **合成演示**：必须带 `data_provenance=synthetic_demo`。仓库样本由固定种子 `20260719` 生成，共 400 条，其中 MINISO 140 条，四个演示竞品各 65 条；不是企业内部数据、真实用户评论、销售记录或市场结论。

## 九个领域方面

`IP/设计吸引力`、`品质/耐用性`、`价格/价值`、`实用性`、`礼赠性`、`收藏性`、`包装`、`门店可得性`、`本地化`。

新增同义词或语言词项时，必须同时补充正向、负向、否定处理和回归测试；不能因为一个词同时出现在方面词典和情感词典，就跳过上下文核验。

## 标准流程

1. 将每条记录规范为 `Evidence`，保留 `source_id`、来源类型、品牌、商品、评分、文本、日期、URL 与 `data_provenance`。
2. 按 `category=interest_goods` 精确过滤，并把目标品牌、配置竞品和其他来源分开。
3. 使用确定性 ABSA 统计每个方面的提及量、覆盖率、负面率、重要性、满意度与代表证据。
4. 计算机会分并稳定排序，机会结论必须关联可解析的 `evidence_ids`。
5. 把高机会项提供给 VOC 候选路径；趋势与白空间路径使用各自证据，避免三个候选收敛为同一概念。
6. 输出前校验引用存在、数据来源标签完整、合成数据免责声明可见。

## 输出契约

输出 `VocReport`，至少包含：

- `review_count`：本次参与分析的目标品牌记录数；
- `aspects`：九方面统计、确定性分数与代表证据；
- `opportunities`：带来源、理由和证据 ID 的排序机会；
- `experience_trend`：在日期信息足够时的方面变化，不足时明确不作时间趋势推断；
- `ost`：结果、机会、候选解法和待验证实验的结构化映射。

## 禁止事项

- 不把合成评论称为真实 VOC、真实用户研究或企业数据。
- 不把离线接受度、NPS、候选总分解释为试购、销量、爆款概率或 ROI。
- 不让大模型自由改写统计数值、权重、阈值或严重风险闸口。
- 不为缺失来源的观点伪造证据 ID，不用公司经营规模替代单品需求证据。
